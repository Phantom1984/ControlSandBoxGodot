# 02 Welch 法功率谱估计

## 1. 为什么需要 Welch 法

直接周期图（Periodogram）的方差很大，且不随数据长度增加而减小（不收敛）。Welch 法通过**分段加窗平均**显著降低方差，是工程上估计功率谱/互谱的标准方法。

| 方法 | 方差 | 偏差 | 频率分辨率 | 备注 |
|------|------|------|----------|------|
| 周期图 | 大 | 小 | 高 | 不收敛 |
| Bartlett | 小 | 中 | 中 | 分段不重叠不加窗 |
| **Welch** | 更小 | 中 | 中 | 分段 + 加窗 + 重叠 |
| BT 法 | 小 | 大 | 低 | 自相关 + 窗 |

## 2. Welch 算法流程

给定信号 `x[n]`，长度 `N`：

1. **分段**：每段长 `L`，相邻段重叠 `D`（典型 `D = L/2`）
   - 段数 `K = ⌊(N - L) / (L - D)⌋ + 1`
2. **加窗**：每段乘窗函数 `w[n]`（Hann / Hamming / Blackman）
3. **FFT**：每段做 `L` 点 FFT
   `Xk(f) = Σ xk[n] · w[n] · e^(-j2πfn/L)`
4. **修正周期图**：
   `Pk(f) = |Xk(f)|² / (U · fs)`，其中 `U = Σ w²[n]`
5. **平均**：
   `Ŝxx(f) = (1/K) · Σ Pk(f)`

**互谱估计**：把第 4 步的 `|Xk|²` 替换为 `Xk*(f) · Yk(f)` 后平均。

## 3. 窗函数选择

| 窗 | 主瓣宽 | 旁瓣电平 | 适用场景 |
|----|-------|---------|---------|
| 矩形 | 最窄 | -13 dB | 分离相近频率 |
| Hann | 2 倍 | -31 dB | 通用首选 |
| Hamming | 2 倍 | -43 dB | 旁瓣要求低 |
| Blackman | 3 倍 | -58 dB | 强干扰环境 |

**默认 Hann 窗**：旁瓣足够低、主瓣不过宽，频域辨识场景首选。

## 4. 关键权衡：分辨率 vs 方差

频率分辨率：
```
Δf = fs / L
```

- `L` 大 → 分辨率高，但段数 `K` 减少 → 方差大
- `L` 小 → 段数多 → 方差小，但分辨率下降

**经验法则**：
- 共振峰半功率带宽 > 5·Δf 才能可靠分辨
- 段数 `K ≥ 8` 才有统计意义
- 高 Q 系统（如倒立摆）需较大 `L`，但样本量受限时要折中

## 5. 重叠率

- 不重叠：信息利用不充分，方差大
- 50% 重叠：Hann 窗下的标准选择，相邻段相关性适中
- 75% 重叠：方差进一步降低，但计算量翻倍

Welch 经典论文证明 Hann 窗 50% 重叠下，方差-计算量比最优。

## 6. 归一化与功率保持

加窗会损失能量，必须补偿：
```
U = Σ w²[n]
```
最终 PSD 估计除以 `U · fs`，单位才是正确的功率谱密度（`V²/Hz`）。

互谱同理，但保留复数形式。

## 8. 关键参数速查

| 参数 | 典型值 | 影响 |
|------|-------|------|
| 段长 `L` | 256 / 512 / 1024 | 决定 Δf |
| 重叠 `D` | `L/2` | 方差-计算量权衡 |
| 窗 | Hann | 默认 |
| 段数 `K` | ≥ 8 | 越多方差越小 |
| FFT 点数 | ≥ `L`（可零填充） | 频率插值密度（不提高真实分辨率） |
