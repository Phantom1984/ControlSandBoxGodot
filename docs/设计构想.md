# ControlSandBox 设计构想

## 一、项目定位

一个基于物理引擎的控制算法沙盒。用户只需关注**控制器部分**的设计与搭建，被控对象的动力学由物理引擎真实仿真，从而在接近真实的环境中验证控制算法的可行性。

与 Simulink 的区别：Simulink 需要对整个系统（含被控对象）建模，而本工具只针对控制器部分，被控对象通过拖拽部件搭建结构后由物理引擎驱动。

## 二、核心功能

### 1. 控制器设计
- 预设常用控制器模块（PID、LQR、LPF 等），支持拖拽快速使用
- 支持用户自定义控制器结构框图，搭建：
  - 开环控制 / 闭环控制
  - 双环控制（如内环摆角 + 外环位置）
  - 前馈控制
  - 滤波器组合等
- 使用前需先选择小车采集的数据（速度、位置等），输出为力，中间环节即为控制器框图

### 2. 被控对象搭建
- 支持用户拖拽部件，自行搭建被控对象的机械结构

### 3. 外部扰动与激励
- 施加摩擦、空气阻力、周期性扰动、脉冲冲量等外部激励
- 支持系统辨识实验

### 4. 环境参数设置
- 可调节重力、摩擦系数、阻力系数等环境参数

### 5. 数据分析与可视化
- 时域响应图
- 伯德图（Bode Plot）
- Scope 示波器功能，实时监控输出结果

## 四、后续规划

### 控制算法
- **LQR 补齐**：完成倒立摆状态方程线性化与 Riccati 方程求解（迭代法或硬编码 K）
- **滤波器仿真**：LowPass、Notch、Biquad 等
- **前馈控制结构**：基于辨识得到的模型设计前馈补偿
- **MPC / SMC**：模型预测控制、滑模控制等进阶算法

### 被控对象
- **部件拖拽搭建**：用户自行组装被控对象机械结构（当前为预设 Cart / PendulumCart）
- **二阶倒立摆**：双摆杆系统

### 辨识与数据分析
- **时域辨识 ARX/ARMAX**：最小二乘参数估计、阶次确定
- **数据导出**：将采集的 u-y 序列导出为 CSV / JSON，便于离线分析
- **模型验证**：辨识得到的传递函数与实测数据对比验证

### 交互与体验
- **控制器框图拖拽编辑**：可视化搭建控制器结构
- **多种输入源**：阶跃、正弦、方波、自定义信号（控制场景）
- **更多时域 / 频域分析工具**：阶跃响应指标、 Nyquist 图等
